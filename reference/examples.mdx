This page provides practical, real-world examples of using fzf to enhance your command-line workflows.

## File Operations

### Interactive File Finder

Open files in your editor with fuzzy search and preview:

<CodeGroup>
```bash Basic
# Open file in vim
vim $(fzf)
```

```bash With Preview
# Open with preview using bat
fzf --preview 'bat --color=always --style=numbers {}' \
    --bind 'enter:become(vim {})'
```

```bash Advanced
# Multi-select with preview and custom binding
fzf --multi \
    --preview 'bat --color=always --style=numbers {}' \
    --preview-window 'right:60%:wrap' \
    --bind 'ctrl-/:change-preview-window(down|hidden|)' \
    --bind 'ctrl-y:execute-silent(echo {} | pbcopy)' \
    --bind 'enter:become(vim {+})'
```
</CodeGroup>

### Directory Navigator

Quickly navigate to any directory:

```bash
# Find and cd to directory
cd $(find . -type d | fzf)

# With tree preview
cd $(find . -type d | fzf \
    --preview 'tree -C {} | head -100' \
    --preview-window right:50%)

# Using fd (faster)
cd $(fd --type d | fzf --preview 'eza --tree --level=2 --color=always {}')
```

### File Search and Edit

Search file contents and open at the matching line:

```bash
#!/usr/bin/env bash
# Save as: fzf-edit

rg --color=always --line-number --no-heading --smart-case "${*:-}" |
  fzf --ansi \
      --delimiter : \
      --preview 'bat --color=always {1} --highlight-line {2}' \
      --preview-window 'up,60%,border-bottom,+{2}+3/3,~3' \
      --bind 'enter:become(vim {1} +{2})'
```

## Process Management

### Interactive Process Killer

Kill processes interactively:

<CodeGroup>
```bash Simple
# Kill single process
ps aux | fzf | awk '{print $2}' | xargs kill -9
```

```bash With Reload
# With ability to reload process list
(date; ps aux) |
  fzf --bind='ctrl-r:reload(date; ps aux)' \
      --header=$'Press CTRL-R to reload\n\n' \
      --header-lines=2 \
      --preview='echo {}' \
      --preview-window=down,3,wrap \
      --layout=reverse \
      --height=80% |
  awk '{print $2}' | xargs kill -9
```

```bash Multi-Select
# Kill multiple processes
ps aux | fzf --multi \
    --header 'TAB to select multiple processes' \
    --bind 'enter:become(echo {+} | awk "{{print \$2}}" | xargs kill -9)'
```
</CodeGroup>

### Port Finder

Find and kill processes by port:

```bash
# Find process using a port and kill it
lsof -i -P -n | fzf | awk '{print $2}' | xargs kill -9

# With header
lsof -i -P -n | fzf --header-lines=1 \
    --header 'Select process to kill' \
    --preview 'ps -p {2} -o cmd' \
    --bind 'enter:become(kill -9 {2})'
```

## Git Integration

### Interactive Git File Selector

Work with git-tracked files:

```bash

git status --short | fzf --preview 'git diff --color=always {2}' |
  awk '{print $2}' | xargs vim

# Stage files interactively
git status --short | fzf --multi --preview 'git diff --color=always {2}' |
```

### Git Branch Switcher

Quickly switch between branches:

```bash
# Simple branch switcher
git branch | fzf | sed 's/^[* ]*//' | xargs git checkout

# With commit preview
git branch --all --format='%(refname:short)' |
  fzf --preview 'git log --oneline --color=always {}' \
      --preview-window right:70% \
      --bind 'enter:become(git checkout {})'
```

Browse and show commits:

```bash
      --preview-window right:60% \

# Copy commit hash

  fzf --ansi \
      --bind 'ctrl-y:execute-silent(echo {1} | pbcopy)+abort' \
```

## Command History

### Enhanced History Search

Custom history search with better preview:

```bash
# Add to .bashrc or .zshrc
fzf-history() {
  eval $(history | fzf --tac --no-sort \
    --preview 'echo {}' \
    --preview-window down:3:wrap \
    --bind 'ctrl-r:toggle-sort' |
    sed 's/^[ ]*[0-9]*[ ]*//')
}

bind -x '"\C-h": fzf-history'  # bash
# or
bindkey '^h' fzf-history        # zsh
```

## Ripgrep Integration

### Interactive Ripgrep

Search code interactively with ripgrep:

```bash
#!/usr/bin/env bash
# Save as: rg-fzf

RG_PREFIX="rg --column --line-number --no-heading --color=always --smart-case "
INITIAL_QUERY="${*:-}"

fzf --ansi --disabled --query "$INITIAL_QUERY" \
    --bind "start:reload:$RG_PREFIX {q}" \
    --bind "change:reload:sleep 0.1; $RG_PREFIX {q} || true" \
    --delimiter : \
    --preview 'bat --color=always {1} --highlight-line {2}' \
    --preview-window 'up,60%,border-bottom,+{2}+3/3,~3' \
    --bind 'enter:become(vim {1} +{2})'
```

### Dual-Mode Search

Switch between ripgrep and fzf search:

```bash
#!/usr/bin/env bash

rm -f /tmp/rg-fzf-{r,f}
RG_PREFIX="rg --column --line-number --no-heading --color=always --smart-case "
INITIAL_QUERY="${*:-}"

fzf --ansi --disabled --query "$INITIAL_QUERY" \
    --bind "start:reload:$RG_PREFIX {q}" \
    --bind "change:reload:sleep 0.1; $RG_PREFIX {q} || true" \
    --bind "ctrl-f:unbind(change,ctrl-f)+change-prompt(2. fzf> )+enable-search+rebind(ctrl-r)+transform-query(echo {q} > /tmp/rg-fzf-r; cat /tmp/rg-fzf-f)" \
    --bind "ctrl-r:unbind(ctrl-r)+change-prompt(1. ripgrep> )+disable-search+reload($RG_PREFIX {q} || true)+rebind(change,ctrl-f)+transform-query(echo {q} > /tmp/rg-fzf-f; cat /tmp/rg-fzf-r)" \
    --color "hl:-1:underline,hl+:-1:underline:reverse" \
    --prompt '1. ripgrep> ' \
    --delimiter : \
    --header 'CTRL-R (ripgrep) / CTRL-F (fzf)' \
    --preview 'bat --color=always {1} --highlight-line {2}' \
    --preview-window 'up,60%,border-bottom,+{2}+3/3,~3' \
    --bind 'enter:become(vim {1} +{2})'
```

## Docker Integration

### Container Manager

Manage Docker containers interactively:

```bash
# Stop container
docker ps | fzf --header-lines=1 | awk '{print $1}' | xargs docker stop

# View logs
docker ps | fzf --header-lines=1 \
    --preview 'docker logs --tail 100 {1}' \
    --preview-window up:60%:follow \
    --bind 'enter:become(docker logs -f {1})'

# Exec into container
docker ps | fzf --header-lines=1 | awk '{print $1}' | xargs -I {} docker exec -it {} bash
```

### Image Manager

Work with Docker images:

```bash
# Remove image
docker images | fzf --header-lines=1 | awk '{print $3}' | xargs docker rmi

# Run container from image
docker images | fzf --header-lines=1 \
    --preview 'docker history {3}' \
    --bind 'enter:execute(docker run -it {3} bash)+abort'
```

```bash
      --preview 'kubectl logs --follow --all-containers --tail=100 --namespace {1} {2}'
```

## Shell Functions

Useful shell functions that combine fzf with common tasks:

### Change Directory to Recent

```bash
# Add to .bashrc or .zshrc
cd-recent() {
  local dir
  dir=$(dirs -v | fzf --height 40% --reverse | awk '{print $2}')
  [ -n "$dir" ] && cd "$dir"
}

alias cdr='cd-recent'
```

### Find and Replace

```bash
# Interactive find and replace
fzf-replace() {
  local file
  file=$(rg -l "$1" | fzf --preview "rg --color=always '$1' {}")
  [ -n "$file" ] && vim "+/$1" "$file"
}
```

### Project Switcher

```bash
# Switch between project directories
project() {
  local project_dir
  project_dir=$(fd --type d --max-depth 2 . ~/projects | fzf \
    --preview 'eza --tree --level=2 --color=always {}' \
    --preview-window right:60%)
  [ -n "$project_dir" ] && cd "$project_dir"
}
```

### Command Palette

Create a custom command palette:

```bash
# Create ~/.commands file with format: description | command
# Example:
# Open vim config | vim ~/.vimrc
# Update system | sudo apt update && sudo apt upgrade

commands() {
  local cmd
  cmd=$(cat ~/.commands | fzf --delimiter '|' --preview 'echo {2}' --with-nth 1 | awk -F'|' '{print $2}')
  [ -n "$cmd" ] && eval "$cmd"
}
```

## Tips and Best Practices

<Tip>
**Use `become` instead of command substitution**: `fzf --bind 'enter:become(vim {})'` is more robust than `vim $(fzf)` as it handles empty results and special characters better.
</Tip>

<Tip>
**Debounce rapid changes**: When using `reload` with `change` event, add a small delay: `change:reload:sleep 0.1; command {q}`
</Tip>

<Tip>
**Preview with offset**: Use `+{N}+OFFSET/DENOM` to scroll preview to relevant line: `--preview-window '+{2}+3/3'`
</Tip>

<Tip>
**Multi-select with `{+}`**: Use `{+}` placeholder to get all selected items: `--bind 'enter:become(vim {+})'`
</Tip>

<Warning>
Always quote file paths and handle special characters properly when using `xargs` or command substitution. Consider using `--print0` with `xargs -0` for safer handling.
</Warning>

<Tip>
**Context-specific options**: Use different fzf options for different use cases rather than putting everything in `FZF_DEFAULT_OPTS`.
</Tip>
