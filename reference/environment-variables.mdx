---
title: Environment Variables
description: Configure fzf behavior using environment variables for default commands and options
---

fzf can be configured using environment variables to set default behavior across all invocations. This is particularly useful for customizing your shell integration and default options.

## Core Environment Variables

### FZF_DEFAULT_COMMAND

Defines the default command to use when fzf input is a tty (not piped).

<ParamField path="FZF_DEFAULT_COMMAND" type="string">
  Default command to generate the list of candidates
  
  **Example:**
  ```bash
  # Use fd instead of find for faster file searching
  export FZF_DEFAULT_COMMAND='fd --type f --strip-cwd-prefix'
  
  # Use ripgrep to respect .gitignore
  export FZF_DEFAULT_COMMAND='rg --files --hidden --follow --glob "!.git/*"'
  ```
</ParamField>

<Tip>
When `FZF_DEFAULT_COMMAND` is not set, fzf will use its built-in file traversal using the `--walker` options.
</Tip>

### FZF_DEFAULT_OPTS

Defines default command-line options to be applied to every fzf invocation.

<ParamField path="FZF_DEFAULT_OPTS" type="string">
  Default options (e.g., `--layout=reverse --info=inline`)
  
  **Examples:**
  ```bash
  # Basic setup with reverse layout
  export FZF_DEFAULT_OPTS="--layout=reverse --info=inline"
  
  # More comprehensive setup
  export FZF_DEFAULT_OPTS="
    --height 40%
    --layout reverse
    --border
    --info inline
    --margin 1
    --padding 1
    --preview 'bat --style=numbers --color=always {}'
    --preview-window right:50%
    --color 'border:#6699cc'
  "
  
  # With tmux integration
  export FZF_DEFAULT_OPTS="--height 40% --tmux bottom,40% --layout reverse --border top"
  ```
</ParamField>

<Warning>
Do not add `--preview` with file-specific commands to `FZF_DEFAULT_OPTS` if you plan to use fzf with non-file inputs like process lists or command history. The preview command should match your input type.
</Warning>

### FZF_DEFAULT_OPTS_FILE

Specifies a file containing default options, useful for managing complex configurations.

<ParamField path="FZF_DEFAULT_OPTS_FILE" type="string">
  Location of the file to read default options from
  
  **Example:**
  ```bash
  export FZF_DEFAULT_OPTS_FILE=~/.fzfrc
  ```
  
  Then create `~/.fzfrc` with your options:
  ```bash
  --height 40%
  --layout reverse
  --border rounded
  --info inline
  --preview 'bat --style=numbers --color=always {}'
  --color 'fg:#bbccdd,bg:#1e1e2e,border:#6699cc'
  ```
</ParamField>

### FZF_API_KEY

Security key for the HTTP server when using `--listen`.

<ParamField path="FZF_API_KEY" type="string">
  X-API-Key header for HTTP server authentication
  
  Used when running fzf with the `--listen` option to accept remote commands.
  
  **Example:**
  ```bash
  export FZF_API_KEY="your-secret-key"
  fzf --listen 6266
  ```
</ParamField>

## Shell Integration Variables

These variables customize the behavior of fzf's shell integration key bindings.

### CTRL-T Integration

Customize the `CTRL-T` key binding for inserting files/directories.

<ParamField path="FZF_CTRL_T_COMMAND" type="string">
  Command to generate the list for `CTRL-T`
  
  **Default:** Uses fzf's built-in `--walker file,dir,follow,hidden`
  
  **Examples:**
  ```bash
  # Use fd for faster traversal
  export FZF_CTRL_T_COMMAND='fd --type f --type d'
  
  # Use find with custom options
  export FZF_CTRL_T_COMMAND='find . -type f -o -type d'
  
  # Disable CTRL-T binding
  export FZF_CTRL_T_COMMAND=''
  ```
</ParamField>

<ParamField path="FZF_CTRL_T_OPTS" type="string">
  Additional fzf options for `CTRL-T`
  
  **Examples:**
  ```bash
  # Preview files with bat
  export FZF_CTRL_T_OPTS="
    --walker-skip .git,node_modules,target
    --preview 'bat -n --color=always {}'
    --bind 'ctrl-/:change-preview-window(down|hidden|)'
  "
  
  # With tree preview for directories
  export FZF_CTRL_T_OPTS="
    --preview '[[ -f {} ]] && bat --color=always {} || tree -C {}'
    --preview-window right:60%
  "
  ```
</ParamField>

<Accordion title="Example: Complete CTRL-T Setup">
```bash
# Use fd for file discovery
export FZF_CTRL_T_COMMAND='fd --type f --type d --hidden --follow --exclude .git'

# Configure preview and shortcuts
export FZF_CTRL_T_OPTS="
  --walker-skip .git,node_modules,target
  --preview 'bat -n --color=always --line-range :500 {}'
  --preview-window 'right:60%:wrap'
  --bind 'ctrl-/:change-preview-window(down|hidden|)'
  --bind 'ctrl-y:execute-silent(echo {} | pbcopy)'
  --header 'CTRL-T: Files | CTRL-/: Toggle preview | CTRL-Y: Copy path'
"
```
</Accordion>

### CTRL-R Integration

Customize the `CTRL-R` key binding for command history search.

<ParamField path="FZF_CTRL_R_COMMAND" type="string">
  Reserved for future use
  
  Currently, setting this to an empty string disables the `CTRL-R` binding.
  
  ```bash
  # Disable CTRL-R binding
  export FZF_CTRL_R_COMMAND=''
  ```
</ParamField>

<ParamField path="FZF_CTRL_R_OPTS" type="string">
  Additional fzf options for `CTRL-R`
  
  **Examples:**
  ```bash
  # Copy command to clipboard
  export FZF_CTRL_R_OPTS="
    --bind 'ctrl-y:execute-silent(echo -n {2..} | pbcopy)+abort'
    --color header:italic
    --header 'Press CTRL-Y to copy command into clipboard'
  "
  
  # With custom preview
  export FZF_CTRL_R_OPTS="
    --preview 'echo {}'
    --preview-window down:3:wrap
    --bind 'ctrl-/:toggle-preview'
  "
  ```
</ParamField>

<Accordion title="Example: Enhanced CTRL-R Setup">
```bash
export FZF_CTRL_R_OPTS="
  --preview 'echo {}' 
  --preview-window down:3:hidden:wrap
  --bind 'ctrl-/:toggle-preview'
  --bind 'ctrl-y:execute-silent(echo -n {2..} | pbcopy)+abort'
  --color header:italic
  --header 'CTRL-Y: Copy | CTRL-/: Preview | ALT-R: Toggle raw mode'
"
```
</Accordion>

### ALT-C Integration

Customize the `ALT-C` key binding for changing directories.

<ParamField path="FZF_ALT_C_COMMAND" type="string">
  Command to generate the list for `ALT-C`
  
  **Default:** Uses fzf's built-in `--walker dir,follow,hidden`
  
  **Examples:**
  ```bash
  # Use fd to find directories
  export FZF_ALT_C_COMMAND='fd --type d'
  
  # Use find with custom depth limit
  export FZF_ALT_C_COMMAND='find . -type d -maxdepth 3'
  
  # Disable ALT-C binding
  export FZF_ALT_C_COMMAND=''
  ```
</ParamField>

<ParamField path="FZF_ALT_C_OPTS" type="string">
  Additional fzf options for `ALT-C`
  
  **Examples:**
  ```bash
  # Preview directory structure with tree
  export FZF_ALT_C_OPTS="
    --walker-skip .git,node_modules,target
    --preview 'tree -C {}'
  "
  
  # With eza for colorful directory listing
  export FZF_ALT_C_OPTS="
    --preview 'eza --tree --level=2 --color=always {}'
    --preview-window right:60%
  "
  ```
</ParamField>

<Accordion title="Example: Complete ALT-C Setup">
```bash
# Use fd for directory discovery
export FZF_ALT_C_COMMAND='fd --type d --hidden --follow --exclude .git'

# Configure preview
export FZF_ALT_C_OPTS="
  --walker-skip .git,node_modules,target
  --preview 'eza --tree --level=2 --color=always {} 2>/dev/null || tree -C {} | head -200'
  --preview-window right:60%
  --header 'ALT-C: Change directory'
"
```
</Accordion>

## Fuzzy Completion Variables

These variables control the behavior of fuzzy completion (the `**<TAB>` trigger).

### FZF_COMPLETION_TRIGGER

<ParamField path="FZF_COMPLETION_TRIGGER" type="string" default="**">
  Trigger sequence for fuzzy completion
  
  **Example:**
  ```bash
  # Use ~~ as trigger instead of **
  export FZF_COMPLETION_TRIGGER='~~'
  
  # Usage: vim ~~<TAB>
  ```
</ParamField>

### FZF_COMPLETION_OPTS

<ParamField path="FZF_COMPLETION_OPTS" type="string">
  Default options for fuzzy completion
  
  **Example:**
  ```bash
  export FZF_COMPLETION_OPTS="
    --border
    --info=inline
    --preview 'bat --color=always --style=numbers {}'
  "
  ```
</ParamField>

### FZF_COMPLETION_PATH_OPTS

<ParamField path="FZF_COMPLETION_PATH_OPTS" type="string">
  Options for path completion (e.g., `vim **<TAB>`)
  
  **Example:**
  ```bash
  export FZF_COMPLETION_PATH_OPTS="
    --walker file,dir,follow,hidden
    --walker-skip .git,node_modules
  "
  ```
</ParamField>

### FZF_COMPLETION_DIR_OPTS

<ParamField path="FZF_COMPLETION_DIR_OPTS" type="string">
  Options for directory completion (e.g., `cd **<TAB>`)
  
  **Example:**
  ```bash
  export FZF_COMPLETION_DIR_OPTS="
    --walker dir,follow
    --preview 'tree -C {} | head -100'
  "
  ```
</ParamField>

<Accordion title="Example: Complete Fuzzy Completion Setup">
```bash
# Custom trigger
export FZF_COMPLETION_TRIGGER='~~'

# General completion options
export FZF_COMPLETION_OPTS="
  --border
  --info=inline
"

# Path completion (vim ~~<TAB>)
export FZF_COMPLETION_PATH_OPTS="
  --walker file,dir,follow,hidden
  --walker-skip .git,node_modules,target
  --preview 'bat --color=always --style=numbers {}'
"

# Directory completion (cd ~~<TAB>)
export FZF_COMPLETION_DIR_OPTS="
  --walker dir,follow
  --preview 'eza --tree --level=2 --color=always {}'
"
```
</Accordion>

## Complete Configuration Example

Here's a comprehensive example combining all environment variables:

<CodeGroup>
```bash Bash/Zsh
# ~/.bashrc or ~/.zshrc

# Core fzf configuration
export FZF_DEFAULT_COMMAND='fd --type f --strip-cwd-prefix --hidden --follow --exclude .git'
export FZF_DEFAULT_OPTS="
  --height 40%
  --layout reverse
  --border rounded
  --info inline
  --margin 1
  --padding 1
  --tmux bottom,40%
  --color 'fg:#bbccdd,bg:#1e1e2e,border:#6699cc'
  --color 'hl:#bd93f9,fg+:#f8f8f2,bg+:#44475a,hl+:#bd93f9'
  --bind 'ctrl-/:toggle-preview'
  --bind 'ctrl-y:execute-silent(echo {} | pbcopy)'
"

# Shell integration: CTRL-T (Files/Directories)
export FZF_CTRL_T_COMMAND='fd --type f --type d --hidden --follow --exclude .git'
export FZF_CTRL_T_OPTS="
  --walker-skip .git,node_modules,target
  --preview 'bat -n --color=always --line-range :500 {}'
  --preview-window right:60%:wrap
  --header 'CTRL-/: Toggle preview | CTRL-Y: Copy path'
"

# Shell integration: CTRL-R (Command history)
export FZF_CTRL_R_OPTS="
  --preview 'echo {}' 
  --preview-window down:3:hidden:wrap
  --bind 'ctrl-y:execute-silent(echo -n {2..} | pbcopy)+abort'
  --header 'CTRL-Y: Copy command | CTRL-/: Toggle preview'
"

# Shell integration: ALT-C (Change directory)
export FZF_ALT_C_COMMAND='fd --type d --hidden --follow --exclude .git'
export FZF_ALT_C_OPTS="
  --walker-skip .git,node_modules,target
  --preview 'tree -C {} | head -100'
  --header 'ALT-C: Change directory'
"

# Fuzzy completion
export FZF_COMPLETION_TRIGGER='**'
export FZF_COMPLETION_OPTS='--border --info=inline'

# Initialize shell integration
eval "$(fzf --bash)"  # or: source <(fzf --zsh) for zsh
```

## Tips and Best Practices

<Tip>
**Performance**: Use `fd` or `rg` instead of `find` for `FZF_DEFAULT_COMMAND` for significantly better performance on large directories.
</Tip>

<Tip>
**Modularity**: Keep complex configurations in `FZF_DEFAULT_OPTS_FILE` for easier management and version control.
</Tip>

<Warning>
**Context-Specific Previews**: Avoid adding file-specific preview commands to `FZF_DEFAULT_OPTS`. Use context-specific variables like `FZF_CTRL_T_OPTS` instead.
</Warning>

<Tip>
**Testing**: Test your configuration changes by running fzf directly before adding them to your shell configuration file.
</Tip>
