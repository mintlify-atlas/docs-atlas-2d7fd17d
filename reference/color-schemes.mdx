fzf provides extensive color customization options through the `--color` option. You can choose from base color schemes and customize individual UI elements to create your perfect theme.

## Base Color Schemes

fzf includes several built-in base color schemes optimized for different terminal backgrounds.

### Dark Theme

The default dark color scheme with 256-color support.

```bash
fzf --color=dark
```

Best for terminals with dark backgrounds. Provides good contrast and readability.

### Light Theme

Optimized for light terminal backgrounds.

```bash
fzf --color=light
```

Ideal for light-themed terminals, with darker text on lighter backgrounds.

### Base16 Theme

Uses only the 16 base terminal colors.

```bash
fzf --color=base16
# or
fzf --color=16
```

Respects your terminal's color scheme configuration, making fzf blend seamlessly with your terminal theme.

### No Color (Black & White)

Disables all colors for a monochrome display.

```bash
fzf --color=bw
# or
fzf --color=no
# or
fzf --no-color
```

Useful for terminals that don't support colors or when you prefer a minimal look.

<Tip>
You can also disable colors by setting the `NO_COLOR` environment variable, which fzf respects automatically.
</Tip>

## Color Elements

You can customize individual UI elements by specifying them after the base scheme.

### Format

```bash
fzf --color="ELEMENT:COLOR[:ATTRIBUTES]"
```

### Available Elements

<Accordion title="Text Elements">
- `fg` - Foreground color
- `bg` - Background color
- `input`, `input-fg` - Input text color
- `query` - Alias for input-fg
- `ghost` - Ghost text (placeholder)
- `disabled` - Disabled text
- `hl` - Highlighted substrings
- `current-fg`, `fg+` - Current line foreground
- `current-bg`, `bg+` - Current line background
- `current-hl`, `hl+` - Current line highlighted substring
- `nth` - nth field color
- `nomatch` - No match indicator
</Accordion>

<Accordion title="List Elements">
- `list-fg` - List foreground color
- `list-bg` - List background color
- `selected-fg` - Selected item foreground
- `selected-bg` - Selected item background
- `selected-hl` - Selected item highlighted substring
- `alt-bg` - Alternate row background
- `gutter` - Gutter area
- `alt-gutter` - Alternate gutter
</Accordion>

<Accordion title="UI Elements">
- `border` - Main border
- `list-border` - List section border
- `input-border` - Input section border
- `preview-border` - Preview window border
- `header-border` - Header section border
- `footer-border` - Footer section border
- `separator` - Separator line
- `scrollbar` - Scrollbar
- `preview-scrollbar` - Preview scrollbar
- `gap-line` - Gap line between items
</Accordion>

<Accordion title="Labels and Indicators">
- `label` - Border label (alias for border-label)
- `border-label` - Main border label
- `list-label` - List border label
- `input-label` - Input border label
- `preview-label` - Preview border label
- `header-label` - Header border label
- `footer-label` - Footer border label
- `prompt` - Input prompt
- `pointer` - Selection pointer
- `marker` - Multi-select marker
- `spinner` - Loading spinner
- `info` - Info text
</Accordion>

<Accordion title="Preview Window">
- `preview-fg` - Preview text color
- `preview-bg` - Preview background
</Accordion>

<Accordion title="Header and Footer">
- `header`, `header-fg` - Header text
- `header-bg` - Header background
- `footer`, `footer-fg` - Footer text
- `footer-bg` - Footer background
</Accordion>

### Color Specification

Colors can be specified in several formats:

<ParamField path="Named Colors" type="string">
  Use ANSI color names:
  
  - `black`, `red`, `green`, `yellow`, `blue`, `magenta`, `cyan`, `white`
  - `bright-black` (or `gray`/`grey`)
  - `bright-red`, `bright-green`, `bright-yellow`, `bright-blue`, `bright-magenta`, `bright-cyan`, `bright-white`
  
  ```bash
  fzf --color="fg:blue,bg:black"
  ```
</ParamField>

<ParamField path="ANSI Color Numbers" type="number">
  Use color numbers (0-255) for 256-color support
  
  ```bash
  fzf --color="fg:15,bg:235"
  ```
  
  - `-1` keeps the original color from input
</ParamField>

<ParamField path="Hex Colors" type="string">
  Use 24-bit true color with hex notation
  
  ```bash
  fzf --color="fg:#bbccdd,bg:#1e1e2e"
  ```
</ParamField>

### Text Attributes

You can add text attributes to color specifications:

- `regular` - Regular text (removes all attributes)
- `bold`, `strong` - Bold text
- `dim` - Dimmed text
- `italic` - Italic text
- `underline` - Underlined text
- `underline-double` - Double underline
- `underline-curly` - Curly underline
- `underline-dotted` - Dotted underline
- `underline-dashed` - Dashed underline
- `blink` - Blinking text
- `reverse` - Reverse video
- `strikethrough` - Strikethrough text
- `strip` - Strip ANSI color codes

**Example:**
```bash
fzf --color="prompt:blue:bold,header:yellow:italic:underline"
```

## Advanced Color Customization

### Combining Base Scheme with Custom Colors

You can start with a base scheme and override specific elements:

```bash
fzf --color=dark,border:#6699cc,prompt:#bd93f9:bold,marker:#50fa7b
```

### Using -1 for Original Colors

Use `-1` to preserve the original color from the input:

```bash
fzf --ansi --color="hl:-1:underline,hl+:-1:underline:reverse"
```

This is particularly useful with `--ansi` to keep colors from tools like ripgrep.

### Context-Aware Colors

Customize colors for different sections:

```bash
fzf --color="fg:#bbccdd,fg+:#ddeeff,bg:#334455" \
    --color="preview-fg:#aabbcc,preview-bg:#223344" \
    --color="border:#778899,preview-border:#667788" \
    --color="list-bg:#445566,list-border:#556677" \
    --color="input-bg:#334455,input-border:#445566"
```

### Semantic Color Usage

<Accordion title="Example: File Browser Theme">
```bash
fzf --walker file,dir,follow,hidden \
    --color="fg:#e0e0e0,bg:#1a1a1a" \
    --color="hl:#ffcc00:bold,hl+:#ffcc00:bold:reverse" \
    --color="border:#404040,border-label:#00ff00:bold" \
    --color="prompt:#00ccff:bold,pointer:#ff6600:bold" \
    --color="marker:#ff0066,spinner:#00ff00" \
    --color="info:#666666:italic" \
    --color="preview-border:#606060" \
    --border-label " Files " \
    --preview 'bat --color=always {}' \
    --preview-window 'border-rounded'
```
</Accordion>

## Creating Your Own Theme

### Step 1: Choose a Base Scheme

Start with a base that matches your terminal background:

```bash
fzf --color=dark  # or light, base16, bw
```

### Step 2: Customize Key Elements

Focus on these essential elements first:

```bash
fzf --color=dark \
    --color="border:#6699cc" \
    --color="prompt:#bd93f9:bold" \
    --color="hl:#ffcc00,hl+:#ffcc00:reverse" \
    --color="pointer:#ff6600:bold"
```

### Step 3: Fine-Tune Details

Add colors for headers, info, and other elements:

```bash
fzf --color=dark \
    --color="border:#6699cc,prompt:#bd93f9:bold" \
    --color="hl:#ffcc00,hl+:#ffcc00:reverse" \
    --color="pointer:#ff6600:bold,marker:#50fa7b" \
    --color="info:#888888:italic,header:#999999:bold" \
    --color="spinner:#00ff00"
```

### Step 4: Test and Iterate

Test your theme with different inputs:

```bash
# Test with files
find . -type f | head -100 | fzf --color="..."

# Test with colored input
ls --color=always | fzf --ansi --color="..."

# Test with preview
fzf --preview 'bat --color=always {}' --color="..."
```

If you use Vim, you can generate fzf colors from your Vim colorscheme:

```vim

  \ 'bg':         ['bg', 'Normal'],
  \ 'preview-bg': ['bg', 'NormalFloat'],
  \ 'hl':         ['fg', 'Comment'],
  \ 'fg+':        ['fg', 'CursorLine', 'CursorColumn', 'Normal'],
  \ 'bg+':        ['bg', 'CursorLine', 'CursorColumn'],
  \ 'hl+':        ['fg', 'Statement'],
  \ 'info':       ['fg', 'PreProc'],

  \ 'header':     ['fg', 'Comment'] }

" Export the color option
:echo fzf#wrap()
```

## Tips and Best Practices

<Tip>
**Contrast is Key**: Ensure sufficient contrast between foreground and background colors for readability.
</Tip>

<Tip>
**Highlight Visibility**: Make sure `hl` and `hl+` colors stand out clearly from regular text.
</Tip>

<Tip>
**Border Consistency**: Use similar colors for related borders (list-border, input-border, preview-border) for visual cohesion.
</Tip>

<Warning>
**Terminal Compatibility**: 24-bit colors (hex format) require a terminal that supports true color. Use ANSI colors (0-255) for better compatibility.
</Warning>

<Tip>
**Preview Background**: Set `preview-bg` to match your terminal background for seamless integration when using tools like bat or cat.
</Tip>

<Tip>
**Test with Content**: Always test your color scheme with real content, including ANSI-colored input if you use `--ansi`.
</Tip>
